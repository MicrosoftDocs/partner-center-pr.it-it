## <a name="readiness-changes-to-the-cloud-solution-provider-csp-customer-address-validation-api-to-go-live-on-july-13-2021"></a><a name="8"></a>Idoneità: le modifiche apportate all'API di convalida dell'indirizzo del cliente Cloud Solution Provider (CSP) verranno salvate il 13 luglio 2021

### <a name="summary"></a>Riepilogo

Le modifiche apportate all'API Convalida indirizzo saranno in tempo reale a partire dal 13 luglio 2021.

### <a name="categories"></a>Categorie

- Data: 09/07/2021
- Preparazione

### <a name="impacted-audience"></a>Destinatari interessati

Cloud Solution Provider i partner con fatturazione diretta (CSP) e i provider indiretti che creano nuovi clienti o aggiornano i dettagli relativi all'indirizzo dei clienti esistenti

### <a name="details"></a>Dettagli

Microsoft Microsoft si impegna a fornire un metodo conforme, sicuro e sicuro per la convalida degli indirizzi dei clienti per la transazione delle sottoscrizioni dei clienti nel programma CSP. A partire dal 13 luglio 2021, le modifiche apportate all'API Convalida indirizzo saranno in tempo reale.
 
Si noti che queste modifiche influiscono solo sull'API Convalida indirizzo. Le API Crea cliente e Aggiorna profilo di fatturazione non sono interessate. Attualmente, l'indirizzo suggerito non deve essere usato con l'API Crea cliente, ma è altamente consigliato.

La risposta restituirà uno dei messaggi di stato seguenti:

| Stato | Descrizione | Numero di indirizzi suggeriti restituiti |
|------------|---------------|------------------|
| Spedizione verificata  | L'indirizzo è verificato e può essere spedito.  | Single  |
| Verified  | L'indirizzo è verificato.  | Single  |
| Interazione obbligatoria  | Gli indirizzi suggeriti sono stati modificati in modo significativo e necessitano di conferma dell'utente.  | Single  |
| Via parziale  | La via specificata nell'indirizzo è parziale e necessita di altre informazioni.  | Multiplo: massimo 3  |
| Locale parziale  | I locali (numero di edificio, numero di suite e altri) sono parziali e necessitano di altre informazioni.  | Multiplo: massimo 3  |
| Multipli  | Esistono più campi che sono parziali nell'indirizzo (potenzialmente includendo anche Street partial e Premises partial).  | Multiplo: massimo 3  |
| Nessuno  | L'indirizzo non è corretto.  | Nessuno  |
| Non convalidato  | Impossibile inviare l'indirizzo tramite il processo di convalida.   | Nessuno  |

Dopo che un indirizzo è stato inviato per la convalida tramite l'API Convalida indirizzo, verrà restituito lo schema di risposta seguente:

```csharp

// <summary>
/// Object represents the address validation response.
/// </summary>

public class AddressValidationResponse
{
   /// <summary>
   /// Gets or sets the original address
   /// </summary>
   /// <value>
   /// Original Address
   /// </value>
   public Address OriginalAddress { get; set; }

   /// <summary>
   /// Gets or sets the suggested addresses
   /// </summary>
   /// <value>
   /// Suggested Addresses
   /// </value>
   public List<Address> SuggestedAddresses { get; set; }

   /// <summary>
   /// Gets or sets the validation status
   /// </summary>
   /// <value>
   /// Status
   /// </value>
   public string Status { get; set; }

   /// <summary>
   /// Gets or sets the validation message
   /// </summary>
   /// <value>
   /// Validation Message
   /// </value>
   public string ValidationMessage { get; set; }

```

Si noti che per il Stati Uniti, la risposta restituirà un suffisso aggiuntivo di quattro cifre per la riga del codice postale se si immettono solo cinque cifre per il codice postale.
Di seguito è riportata una risposta di esempio:

```C#

"suggestedAddress": [
{ 
"country": "US", 
"state": "WA", 
"city": "Redmond", 
"addressLine1": "1 Microsoft Way", 
"postalCode": "98052-8300" 
}
], 
"originalAddress": { 
"country": "US", 
"city": "Redmond", 
"addressLine1": "1 Micro Way", 
"postalCode": "98052" 
}, 
"status": "InteractionRequired", 
"validationMessage": "Address field invalid for property: ‘Street’" 
}
```

## <a name="keep-your-microsoft-certifications-active-with-help-from-partner-center"></a><a name="6"></a>Mantenere attive le certificazioni Microsoft con l'aiuto di Partner Center

### <a name="categories"></a>Categorie

- Data: 09/07/2021
- Far crescere il business

### <a name="impacted-audience"></a>Destinatari interessati

Amministratori partner e amministratori globali che gestiscono lo stato e i rinnovi della certificazione Microsoft

### <a name="details"></a>Dettagli

Mantenere attive le certificazioni Microsoft è ora più semplice, con l'aiuto di Partner Center.

È sempre stato possibile visualizzare le certificazioni dell'organizzazione nel dashboard di Partner Center, ma ora è anche possibile visualizzare le date di scadenza sia nel report sulle competenze utente che nel report di training Partner Center Insights. Questi aggiornamenti hanno il controllo del processo di rinnovo della certificazione per l'organizzazione. Continua a leggere per informazioni su come trovare gli aggiornamenti per la creazione di report e su come trarranno vantaggio.

Nella sezione Competenze del dashboard Partner Center è presente il report sulle competenze. Quando si esportano i dati in Excel, verrà visualizzato il report sulle competenze utente, che fornisce un elenco degli obiettivi ottenuti (ad esempio certificazioni ed esami) che si qualificano per le competenze. Alla fine del report è presente una nuova colonna con etichetta "Skill Expiration Date" (Data di scadenza competenza), in cui sono disponibili le date di scadenza per le certificazioni basate sui ruoli e speciali dei dipendenti.  

Nella sezione Organization Skills (Competenze dell'organizzazione) Partner Center dashboard è possibile trovare il report Partner Center Insights Training. Quando si esportano i dati del report in Excel, è presente una nuova colonna Expiration Date (Data di scadenza), che contiene le date di scadenza per le certificazioni del team.

Questi aggiornamenti consentono di intervenire rapidamente e ricordare ai dipendenti di rinnovare le certificazioni, mantenendo attive sia le credenziali che lo stato del programma MPN.

Come funziona il nuovo processo di rinnovo della certificazione?  

- Invece di andare in un centro di test e pagare per ripetere gli esami di certificazione, i dipendenti visiteranno Microsoft Learn e passeranno una valutazione gratuita del rinnovo della certificazione che li testa sugli ultimi aggiornamenti tecnologici.
- Quando le date di scadenza della certificazione sono vicine, riceveranno notifiche tramite posta elettronica da Microsoft, indirizzandole alla valutazione del rinnovo della certificazione Microsoft Learn.
- I percorsi di apprendimento online contengono materiale di studio per prepararsi alla valutazione, che possono provare tutte le volte che servono.
- Il passaggio della valutazione estende la scadenza della certificazione per un anno.
- Questo processo si ripete ogni anno, assicurando che le competenze siano attuali.

Per allinearsi a questo nuovo processo di rinnovo annuale, Microsoft sta modificando anche l'intervallo di tempo di validità per le certificazioni basate sui ruoli e speciali da due anni a un anno. A partire dal 30 giugno 2021, tutte le certificazioni basate sui ruoli e speciali ottenute saranno valide per un anno.

Si noti che Microsoft Fundamentals e le certificazioni legacy precedenti sono esenti e non sono interessati dal nuovo processo.

### <a name="next-steps"></a>Passaggi successivi

1. Anticipare le date di scadenza per le certificazioni del team: esaminare il report sulle competenze utente o il report di training Partner Center Insights per identificare le date di scadenza della certificazione in arrivo a breve.
2. Intervenire: invitare il team a rinnovare le certificazioni Microsoft prima della data di scadenza.

## <a name="windows-7-extended-security-updates-esu-year-2-2021-product-key-issue-resolved"></a><a name="3"></a>Windows 7 extended security updates (ESU) Year 2 (2021) product key issue resolved

### <a name="categories"></a>Categorie

- Data: 2021-07-02
- Funzionalità
 
### <a name="impacted-audience"></a>Destinatari interessati

Tutti i partner che hanno acquistato Windows 7 aggiornamenti della sicurezza estesa (ESU) anno 2 (2021) tra il 18 maggio 2021 e il 10 giugno 2021

### <a name="details"></a>Dettagli

È stato identificato un problema temporaneo Windows 7 ESU Year 2 (2021). Per gli ordini effettuati tra il 18 maggio 2021 e il 10 giugno 2021 i partner/clienti potrebbero aver ricevuto un codice Product Key per lo SKU anno 1 (2020). Questo problema è stato risolto. Tutti gli ordini effettuati dopo il 10 giugno 2021 avrebbero ricevuto la chiave corretta per Windows 7 ESU Year 2 (2021). Per gli ordini effettuati nel periodo di impatto, i partner devono restituire l'ordine precedente e riacquisiare.

### <a name="categories"></a>Categorie

Cloud Solution Provider partner (CSP), partner dynamics locali e provider di servizi di distribuzione locale

Come parte di questo impegno, gli SKU scontati esistenti vengono ritirati (vedere la scheda SKU ritirati nella promozione di Dynamics 365 Migrazione cloud e il [file](https://partner.microsoft.com/resources/detail/dynamics-365-cloud-promotion-retired-skus-xls)Excel degli SKU ritirati) il 31 luglio 2021 e sostituirli con nuovi SKU di promozione Migrazione cloud Dynamics 365.

- Supporta le migrazioni cloud in più fasi, consentendo alle organizzazioni di aggiungere nuovi utenti tramite il termine della promozione.
